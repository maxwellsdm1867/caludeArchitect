Built-in Tool Selection Guide

Claude Code provides 6 built-in tools. Choosing the wrong one wastes context tokens and produces worse results. The exam tests this decision tree.

TaskCorrect ToolWhy Not the Others
Find files by name or extension Glob — searches file paths Grep searches content, not paths. Bash find is slower and less integrated.
Find text inside files Grep — searches file content Glob only matches file names. Read requires knowing the exact file already.
Read a known file Read — reads file contents Grep finds matches but does not show full file. Bash cat works but is less integrated.
Change a specific section of a file Edit — replaces unique text matches Write replaces the entire file. Bash sed is fragile and error-prone.
Create a new file or rewrite entirely Write — creates/overwrites files Edit requires existing unique text to match. Cannot create from scratch.
Run commands, install packages, execute scripts Bash — executes shell commands Other tools are read/write only. Bash handles process execution.

The key insight for codebase exploration: Grep for content, Glob for paths. Start broad (Glob to find relevant files), then narrow (Grep to find specific patterns), then Read to examine the full context.

💡 Key Insight: Edit requires the old_string to be unique in the file. If your target text appears multiple times, Edit will fail. Either provide more surrounding context to make the match unique, or use the replace_all parameter for global replacements. This is the most common Edit pitfall on the exam.
⚠ Exam Trap: When a question asks “how to find where a function is defined in a large codebase,” the correct answer is Grep (searches file content for the definition pattern). Glob is the wrong answer because it searches file names, not content. Conversely, when the question asks “find all Python test files,” the correct answer is Glob (**/test_*.py), not Grep.
