Unit 1.3 — Structured Output via tool_use & JSON Schemas

Task Statement 4.3 — Enforce structured output using tool use and JSON schemas

⏱ 2 min read → then open the Lab

The tool_choice Decision

tool_use with JSON schemas is the only reliable way to guarantee schema-compliant output. The API validates the response against your schema before returning it — malformed JSON is impossible.

The tool_choice parameter controls whether and how Claude calls tools:

tool_choiceBehaviorUse When
"auto" May return text OR call a tool Model should decide if extraction applies
"any" Must call a tool; picks which one Multiple schemas, unknown document type
{"type":"tool","name":"..."} Must call the specific named tool Single schema, deterministic pipeline

Schema design rule: if a source document might not contain a field, that field must be nullable. Mark it "type": ["string", "null"]. A required field that only accepts non-null values forces the model to fabricate data to satisfy the constraint.

💡 Key Insight: Required field + absent data = fabrication. Make optional fields nullable: "required": ["warranty_expiry"] with "type": ["string", "null"]. The key is always present in output, but the model can return null when the information is not in the source document.
⚠ Exam Trap: "auto" does NOT guarantee a tool call. If the model decides extraction does not apply, it returns conversational text. If you need structured output every time, use "any" to force a tool call.
