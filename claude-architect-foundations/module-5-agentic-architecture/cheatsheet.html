1 — The Agentic Loop (TS 1.1)

stop_reason Control Flow

stop_reasonMeaningYour Code
"tool_use"Model wants toolsExecute, append results, continue
"end_turn"Model is doneBreak — present final response
"max_tokens"Hit token limitHandle truncation

The Correct Pattern

while True → call API → check stop_reason → if "end_turn" break; if "tool_use" execute and continue.

Anti-Patterns (all WRONG)

  • Parsing NL for termination ("I've completed...")
  • Iteration caps as primary stopping mechanism
  • Checking for text content as completion signal
  • Pre-configured step sequences

Key Safety caps (25+) are fine as a safety net, never as the primary mechanism.

2 — Coordinator-Subagent (TS 1.2)

Architecture

ComponentContextTools
CoordinatorFull historyTask tool
SubagentOnly what coordinator passes4–5 domain tools

Exam Rule Subagents do NOT inherit coordinator context

Context must be passed explicitly. This is the #1 tested concept in Domain 1.

Trap More tools = more capable

18 tools on one agent degrades tool selection. Scope to 4–5 per subagent.

Principles

  • Hub-and-spoke: all communication through coordinator
  • No direct subagent-to-subagent communication
  • Parallel dispatch: multiple Task calls in one response
  • Coordinator synthesizes all subagent results

4 — Enforcement & Handoff (TS 1.4)

Decision Tree

Must be 100%? → PROGRAMMATIC (hooks, gates)
Financial/safety? → PROGRAMMATIC
Style/suggestion? → PROMPT guidance

Exam Rule Prompts NEVER guarantee compliance

Non-zero failure rate. For money, safety, compliance → always programmatic.

Structured Handoff

  • Include: actions taken, current state, remaining work, constraints
  • Exclude: transcripts, chain-of-thought, raw API data

6 — Task Decomposition (TS 1.6)

Strategy Selection

Task TypeStrategy
Predictable pipelinePrompt chaining
Variable investigationDynamic decomposition
Multi-file analysisPer-file + cross-file

Per-File + Cross-File

Pass 1: Each file individually (deep local analysis)

Pass 2: All files together (API contracts, data flow, consistency)

Trap Narrow decomposition is always better

FALSE. Overly narrow scoping misses coverage. Example: scoping to "visual arts" misses most of "creative industries."

7 — Session Management (TS 1.7)

Resume vs Fresh Decision

Data changed? → Fresh session + summary
Data unchanged? → Resume (--resume)

Trap --resume is always efficient

FALSE when data has changed. Resumed session operates on stale assumptions.

Fresh Session Summary

  • Key findings (with sources)
  • Approaches tried + results (prevents dead-end repeats)
  • Open questions + remaining work
  • NOT: full conversation history

fork_session

Snapshot of current context for parallel exploration. Both branches inherit understanding.
