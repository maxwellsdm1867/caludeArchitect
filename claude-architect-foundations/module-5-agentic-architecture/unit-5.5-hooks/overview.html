Agent SDK Hook Points

Hooks are code functions that run at specific points in the agentic loop. They intercept, validate, or transform data without modifying the model's behavior via prompts.

HookFires WhenUse Case
PreToolCall Before a tool is executed Validate inputs, block unauthorized calls, enforce prerequisite gates
PostToolUse After a tool returns its result Normalize output (dates, currencies), redact sensitive data, log for audit
PreResponse Before the final response is sent to the user Content filtering, compliance checks on the final output

Hooks vs Prompts: When to Use Which

RequirementHooks (Code)Prompts (Instructions)
Guarantee: must work 100% of the time Use hooks — deterministic Not reliable enough
Data normalization (date formats, currency) Use PostToolUse hook Model may forget or normalize inconsistently
Block unauthorized tool calls Use PreToolCall hook Model may still attempt the call
Style preferences (tone, verbosity) Overkill Use prompts — flexible, easy to iterate
Best practices (suggest alternatives) Overkill Use prompts — non-critical
💡 Key Insight: Hooks execute deterministically in code — they cannot be circumvented by the model. This makes them the correct choice for any rule that must be followed 100% of the time: compliance checks, data normalization, sensitive data redaction, and prerequisite gates. Prompts are for guidance; hooks are for enforcement.
⚠ Exam Trap: When the exam asks about normalizing tool outputs (e.g., ensuring all dates are in ISO format), the answer is a PostToolUse hook, not a prompt instruction. The model might normalize dates correctly 95% of the time via prompts, but a PostToolUse hook does it 100% of the time. For data consistency, always choose the deterministic option.
