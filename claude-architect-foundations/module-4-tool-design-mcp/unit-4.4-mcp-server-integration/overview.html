MCP Configuration: Where Does It Go?

MCP servers are configured in two locations. The exam tests whether you know which to use.

Config FileLocationScopeUse For
.mcp.json Project root Shared (version controlled) Team tooling — all devs get these tools when they clone the repo
~/.claude.json Home directory Personal (not shared) Personal/experimental servers — your own tools, not for the team

Secret Management

PatternExampleVerdict
Environment variable expansion "apiKey": "${API_KEY}" Correct — secrets stay in the environment, config is safe to commit
Hardcoded secrets "apiKey": "sk-ant-abc123..." Wrong — secrets committed to version control
No secrets needed Local filesystem tools No credential configuration required
💡 Key Insight: .mcp.json is for team tooling (shared, version-controlled). ~/.claude.json is for personal servers (not shared). Both are discovered at connection time and available simultaneously. Use ${ENV_VAR} syntax in .mcp.json to reference secrets without committing them — each developer sets the environment variable locally.
⚠ Exam Trap: When a question asks about sharing MCP tools with a team, the answer is always .mcp.json in the project root (version controlled). ~/.claude.json is personal-only. If a distractor suggests putting team tools in ~/.claude.json and telling teammates to copy it, that is always wrong — use .mcp.json with environment variable expansion for secrets.
