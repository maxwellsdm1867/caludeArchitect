The Accuracy Masking Problem

Aggregate accuracy numbers hide field-level failures. The exam tests whether you can identify where to look and how to measure.

Measurement LevelWhat It ShowsWhat It Hides
Aggregate accuracy (97%) Overall system performance looks great Specific fields (dates, amounts) may be at 60%
Per-document-type accuracy Invoices at 99%, handwritten receipts at 45% Which specific fields fail within each type
Per-field accuracy Vendor name: 99%, line items: 72%, tax: 55% Nothing — this is the actionable level

The pattern: always drill down to per-field accuracy on the worst-performing document types. Use stratified sampling (sample by document type, not uniformly) to catch failures that aggregate metrics hide. Build validation sets that represent the hard cases, not just the easy ones.

💡 Key Insight: Field-level confidence scores enable targeted human review. Instead of reviewing all extractions (expensive) or random samples (misses problems), route only low-confidence fields to humans. A document with 95% average confidence but 30% confidence on the "total_amount" field needs human review of that one field, not the entire document.
⚠ Exam Trap: "The system achieves 97% accuracy on the validation set" sounds reassuring but is a distractor signal when the question mentions multiple document types or field categories. The correct response is to break down accuracy by document type and by field — aggregate accuracy masks per-field failures.
