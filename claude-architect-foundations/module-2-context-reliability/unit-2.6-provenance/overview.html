The Source Reliability Decision Table

When synthesizing information from multiple sources, provenance tracking determines whether the output is trustworthy or dangerously unverifiable.

SituationRequired ActionAnti-pattern
Single source, single claim Map claim to source with citation Presenting claim without attribution
Multiple sources agree Note agreement; cite all sources for the claim Citing only one source (hides corroboration)
Sources conflict Report BOTH values with sources; flag the conflict explicitly Silently picking one value or averaging
Temporal data Prioritize most recent source; note date of each Treating all sources as equally current
Coverage gap Explicitly report what was NOT found in any source Omitting unfound information (reader assumes it was checked)

The pattern: every claim in the output must map to a source. Every conflict must be surfaced. Every gap must be reported. The exam tests whether you choose claim-source mappings over unattributed synthesis.

💡 Key Insight: Coverage gap reporting is as important as what you do find. If the user asks about 5 topics and the sources only cover 3, the output must explicitly state "no information found for topics X and Y in any source." Omission without reporting creates a false sense of completeness that readers cannot detect.
⚠ Exam Trap: "Synthesize information from all sources into a coherent summary" sounds professional but is a distractor when sources conflict. Coherent summaries hide conflicts by smoothing over disagreements. The correct answer surfaces conflicts explicitly with claim-source mappings so the reader can evaluate.
