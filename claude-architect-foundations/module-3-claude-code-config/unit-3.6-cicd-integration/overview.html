Claude Code CI/CD Toolkit

Claude Code can run headless in CI pipelines. The exam tests the correct flags and patterns.

Flag / OptionPurposeExample
-p "prompt" Headless (non-interactive) mode — sends a single prompt claude -p "Review this PR for security issues"
--output-format json Machine-readable JSON output for downstream processing claude -p "..." --output-format json
--json Shorthand for --output-format json claude -p "..." --json
--output-format stream-json Streaming JSON for real-time processing claude -p "..." --output-format stream-json
Session isolation Each CI run starts with a fresh context (no prior session) Default behavior in -p mode
--continue Resume most recent session (multi-step CI workflows) claude --continue -p "Now fix the issues you found"

CI/CD Architecture Patterns

PatternWhen to UseImplementation
PR Review Gate Every pull request needs automated review claude -p "Review the diff..." --json → parse findings → post comments
Code Generation Step Generate code as part of build pipeline claude -p "Generate..." --json → write to file → run tests
Multi-Step Pipeline Complex tasks needing prior context Step 1: claude -p "Analyze..."; Step 2: claude --continue -p "Now fix..."
Prior Findings Feed-Forward Feed output from one step as input to the next Capture JSON output, inject into next prompt as context
💡 Key Insight: Session isolation in -p mode is a feature, not a limitation. Each CI run gets a clean context with no contamination from previous runs. This makes results reproducible and prevents accumulated context from biasing reviews. When you need context continuity across steps, explicitly use --continue or pass prior findings as structured input.
⚠ Exam Trap: "Claude Code in CI gives inconsistent results across runs." The wrong answer is "increase temperature" or "use a session cache." The right answer is to ensure session isolation (default with -p) and feed any necessary prior context explicitly via the prompt. Inconsistency usually comes from leaked session state, not from the model itself.
