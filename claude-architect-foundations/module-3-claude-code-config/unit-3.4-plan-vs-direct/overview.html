Unit 3.4 — Plan Mode vs Direct Execution

Task Statement 3.4 — Choose between plan mode and direct execution based on task complexity

⏱ 2 min read → then open the Lab

The Complexity Decision Tree

Claude Code offers plan mode (think before acting) and direct execution (just do it). The exam tests when to use which.

SignalUse Plan ModeUse Direct Execution
Scope Multi-file changes, 10+ files affected Single-file fix, localized change
Clarity Multiple valid approaches, needs exploration Clear stack trace, obvious implementation
Familiarity Unknown codebase, unfamiliar patterns Well-understood code, following existing pattern
Risk Architectural decisions, breaking changes Adding validation, fixing a typo
Prior work Starting from scratch, greenfield Following up on an approved plan

The Plan-Then-Execute Pattern

For complex tasks, the most effective workflow is:

  1. Plan mode — explore the codebase, identify affected files, propose approach
  2. Human review — developer approves, refines, or redirects the plan
  3. Direct execution — implement the approved plan without re-deliberating

The Explore Subagent

FeatureWhat It DoesWhen to Use
Explore subagent Reads files and gathers context without making changes Understanding unfamiliar code before planning
Read-only guarantee Cannot modify files, only observe Safe exploration of production codebases
💡 Key Insight: Plan mode is not "slower but safer" — it is a different cognitive mode. In plan mode, Claude explores broadly and considers tradeoffs. In direct execution, Claude focuses narrowly on implementation. Using plan mode for a simple bug fix wastes time exploring irrelevant code. Using direct execution for an architecture change risks implementing the first idea without considering alternatives.
⚠ Exam Trap: "A developer asks Claude Code to refactor authentication across 15 files and it makes mistakes." The wrong answer is "use a more capable model" or "give more detailed instructions." The right answer is to use plan mode first to explore the codebase and propose an approach, then execute after human approval. The root cause is jumping to execution without understanding the scope.
