Unit 2.1 — Context Preservation Across Long Interactions

Task Statement 5.1 — Manage context windows to preserve critical information across extended conversations

⏱ 2 min read → then open the Lab

The Context Degradation Curve

As conversations extend, Claude's ability to reference earlier details degrades predictably. The exam tests whether you can identify the stage and apply the correct fix.

StageTurnsSymptomFix
1 — Accurate 1–10 References specific class names, variable values, dollar amounts None needed
2 — Vague 10–20 "The pattern we discussed" instead of OrderProcessor.validate() Inject structured fact blocks; use scratchpad files
3 — Generic 20+ "Typical patterns" — context has effectively degraded to zero Delegate to subagent with fresh context; use /compact

The pattern: progressive summarization silently drops numbers, names, and constraints. "$1,234.56 on order #789" becomes "a refund was discussed." The fix is structured fact extraction that preserves critical data in a format the model can reliably reference.

💡 Key Insight: The "lost in the middle" effect means information placed in the middle of long context windows is referenced less accurately than information at the beginning or end. Put case facts blocks at the top of system prompts, not buried in conversation history. When trimming tool output, keep the first and last sections — trim the middle.
⚠ Exam Trap: "Use progressive summarization to keep context manageable" sounds correct but is a distractor when the scenario involves specific numbers, dates, or identifiers. Summarization loses those details. The correct answer preserves them in structured blocks (e.g., a case facts JSON) that survive context compression.
