Module 2: Context Management & Reliability

Domain 5, Task Statements 5.1–5.6 — Claude Architect Foundations Quick Reference

1 — Context Preservation (TS 5.1)

Context Degradation Curve

StageTurnsSymptom
1 Accurate1–10References specific class names, dollar amounts
2 Vague10–20“The pattern we discussed”
3 Generic20+“Typical patterns” — context gone

Fix: Structured Case Facts Block

Inject a JSON block at the top of the system prompt with all critical data: amounts, IDs, dates, names. Instruct model to use exact values.

“Lost in the Middle” Effect

Info at the beginning and end of long contexts is referenced more accurately than info in the middle. Place key data at boundaries.

Trap Progressive Summarization

“$1,234.56 on order #789” becomes “a refund was discussed.” Never the answer when specific numbers must be preserved.

2 — Escalation Patterns (TS 5.2)

Escalation Decision Tree

Customer requests human → Escalate immediately (don’t investigate first)
Policy gap detected → Escalate with context (agent can’t invent policy)
Amount exceeds authority → Escalate (concrete limit, e.g., $500)
N failed attempts → Escalate (define N concretely)
Angry + simple issue → Handle first, offer escalation after
Self-reported low confidence → UNRELIABLE — don’t use

Trap Sentiment-Based Escalation

Sentiment fails both ways: angry-but-simple gets over-escalated; calm-but-complex gets under-escalated. Always a distractor on the exam.

Exam Rule

Correct answer = explicit trigger conditions (observable, deterministic). Wrong answer = sentiment analysis, confidence self-assessment.

3 — Error Propagation (TS 5.3)

The Three Patterns

PatternBehaviorQuality
Silent SuppressionReturns empty as “success”WORST
Catastrophic TerminationException kills entire pipelineBad
Structured PropagationReturns error context + partial resultsCORRECT

Access Failure vs Empty Result

  • Access failure (timeout, permission denied): “we couldn’t look” — retry may help
  • Empty result (no rows, not found): “we looked, nothing there” — retry won’t help

Structured Error Object

{error_type, attempted_query, partial_results, alternatives}

Trap try/except + empty return

“Wrap in try/except, return empty on failure” = silent suppression. Worse than crashing.

4 — Codebase Context (TS 5.4)

Context Management Toolkit

ToolWhenSurvives
Scratchpad fileExternalize decisions before they degradeCompaction, crashes, subagent boundaries
/compactContext window nearly fullN/A — it IS the compression
Subagent delegationScope exceeds single-context capacityEach agent gets fresh full context

/compact is Lossy

Write key decisions to scratchpad before compacting. Read scratchpad after compacting to restore context.

Crash Recovery

Scratchpad files persist on disk. Resume by reading the scratchpad into a fresh session.

Trap “Bigger context window”

Delays degradation but does NOT prevent it. Lost-in-the-middle effect applies even at 200K tokens.

5 — Human Review (TS 5.5)

Accuracy Drill-Down

  1. Aggregate accuracy (97%) — hides everything
  2. Per-document-type (invoices 99%, receipts 45%) — better
  3. Per-field (vendor_name 99%, tax_amount 55%) — actionable

Stratified Sampling

Over-represent hard cases in validation sets. 200 reviews: not 160/30/10 proportional — use 80/60/60 to catch failures in rare types.

Field-Level Confidence Routing

Route individual low-confidence fields to human review, not entire documents. Saves review budget.

Trap “97% accuracy” sounds good

Always ask: 97% on what? Aggregate accuracy with volume-weighted easy documents masks per-field failures on hard document types.

6 — Provenance (TS 5.6)

Claim-Source Mapping Rules

  • Every claim → cite source(s) with date
  • Sources agree → cite all (shows corroboration)
  • Sources conflict → report ALL values, flag conflict
  • Temporal difference → note dates, prefer recent for time-sensitive data
  • Topic not found → explicitly report gap

Coverage Gap Reporting

If user asks about 5 topics and sources cover 3, output must state “no information found for X and Y.” Omission without reporting = false completeness.

Trap “Synthesize into a coherent summary”

Coherent summaries hide conflicts. The exam prefers surfacing disagreements over smoothing them.
