Unit 1.1 — Explicit Criteria over Vague Instructions

Task Statement 4.1 — Design prompts with explicit criteria to improve precision and reduce false positives

⏱ 2 min read → then open the Lab

The Vague-to-Precise Spectrum

Every prompt lands on this spectrum. The exam rewards Level 4.

LevelPrompt StyleOutcome
1 — Useless "Review this code and report any issues." Reports everything from semicolons to architecture opinions
2 — Still Bad "Be conservative and only report high-confidence findings." No actionable criteria; model drops random findings or hedges
3 — Better "Review for bugs and security issues. No style preferences." Excludes style but "bugs" is still undefined
4 — Production DO flag: comment contradicts code behavior.
DON'T flag: imprecise but non-misleading comments.		 Categorical rules eliminate ambiguity at the boundary

The pattern: replace subjective adjectives (careful, conservative, important) with categorical DO/DON'T rules that define exactly what counts as a finding and what to skip.

💡 Key Insight: Explicit categorical criteria (DO flag / DON'T flag) are the only reliable precision control. The model has no calibrated sense of "conservative" in your context. Specific rules about what to flag and what to skip are what the exam rewards.
⚠ Exam Trap: Answer options containing "be conservative," "only report high-confidence findings," or "focus on important issues" are almost always distractors. The correct answer defines explicit categorical rules — not confidence filtering.
