Unit 5.6 — Task Decomposition Strategies

Task Statement 1.6 — Choose between prompt chaining and dynamic decomposition; understand per-file vs cross-file analysis and narrow decomposition risks

⏱ 2 min read → then open the Lab

Two Decomposition Approaches

ApproachHow It WorksBest ForRisk
Prompt chaining
(pre-configured)		 Fixed sequence of steps defined by the developer Well-understood pipelines (extract → validate → format) Cannot adapt to unexpected inputs
Dynamic decomposition
(model-driven)		 Coordinator decides subtasks at runtime based on the input Complex, variable tasks (research, investigation, debugging) Higher latency; requires good coordinator prompt

Per-File + Cross-File Analysis Pattern

For large codebases, reviews, or document sets, decompose into two passes:

PassScopeFocusWhy Both Are Needed
Pass 1: Per-file Each file individually Local issues (bugs, style, logic errors) Deep analysis without attention dilution
Pass 2: Cross-file All files together (or summaries) Integration issues (API contracts, data flow, consistency) Catches issues invisible at the single-file level

Narrow Decomposition Risk

Overly narrow task decomposition can miss coverage. The classic exam example:

ScenarioNarrow DecompositionResult
Research "creative industries" Subagent scoped to "visual arts" only Misses music, film, writing, gaming — incomplete coverage
Analyze security vulnerabilities One subagent per OWASP category Misses cross-category attack chains
💡 Key Insight: Narrow decomposition is not always better. When tasks have overlapping boundaries or require holistic coverage, overly narrow scoping causes the agent to miss important areas. The exam tests this with the "creative industries" example — a subagent scoped to "visual arts" misses most of the creative industries.
⚠ Exam Trap: Answer options suggesting "decompose into the most granular possible subtasks" are often wrong. The correct decomposition level depends on the task. For well-defined pipelines, prompt chaining works. For open-ended research, dynamic decomposition with broader scopes prevents coverage gaps.
