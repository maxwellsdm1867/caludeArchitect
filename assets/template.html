System Prompts

How to set the stage for reliable, role-specific Claude responses.

What Is a System Prompt?

A system prompt is the first message in a conversation that establishes context, role, constraints, and output format for the model. Unlike user messages, the system prompt is typically hidden from end users and persists across the entire conversation.

Effective system prompts follow the GRADE framework: Goal, Role, Audience, Deliverables, Examples.

Choosing System Prompt Complexity

  • Is the task open-ended or constrained?
    • Constrained (e.g., classification)
      • Short system prompt with output schema
    • Open-ended (e.g., writing, analysis)
      • Does the task need domain expertise?
        • Yes → Detailed role + examples
        • No → Concise role + constraints

Exam Trap: The exam may present a system prompt that includes "You must always..." phrasing. Remember: absolute language like "always" and "never" can cause Claude to refuse legitimate edge-case requests. Prefer guidance with escape hatches over rigid mandates.

Key Insight: System prompts are not immutable instructions. Claude treats them as strong guidance, but a well-crafted user message can override them. The exam tests whether you understand the distinction between system-level guidance and hard constraints enforced through tool design.

Code Example

system_prompt.py
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system="You are a senior Python developer reviewing code for "
           "security vulnerabilities. Flag any issues with severity "
           "levels: critical, high, medium, low. Use markdown tables "
           "for your output.",
    messages=[
        {"role": "user", "content": "Review this function:\n\n"
         "def get_user(id):\n"
         "    query = f'SELECT * FROM users WHERE id = {id}'\n"
         "    return db.execute(query)"}
    ]
)

print(message.content[0].text)

Before & After: System Prompts

system = "You are a helpful assistant. Answer questions about Python."

Too vague — no role specificity, no output format, no constraints. Claude will produce generic, unfocused answers. 

system = """You are an expert Python instructor for intermediate developers.
When answering:
- Start with a one-sentence summary
- Show a minimal code example
- Explain edge cases and common pitfalls
- Use type hints in all code
Output format: Markdown with fenced code blocks."""

Specific role, structured output, clear expectations — Claude will deliver consistent, high-quality responses.

Comparison: Prompt Placement

Aspect System Prompt User Message Prefix
Visibility Hidden from end user Visible in conversation
Persistence Entire conversation Single turn only
Token cost Cached after first call Sent every turn
Override risk Low (strong guidance) High (easily overridden)
Best for Role, format, constraints Per-turn instructions

In multi-turn conversations, system prompts remain in context for every exchange. This means overly long system prompts increase cost linearly with conversation length. Strategies to manage this:

  • Layered prompts: Core identity in system, task-specific details in the first user turn.
  • Prompt caching: Use Anthropic's prompt caching to reduce costs on the static system portion.
  • Dynamic injection: Append tool results or context only when relevant, not in every turn.

Knowledge Check

1. Which is the primary advantage of using a system prompt over prepending instructions to the user message?

B) System prompts are sent once and cached by the API. They persist for the entire conversation, so you avoid re-sending role/format instructions on every turn. Option C is wrong because system prompts can be overridden by sufficiently specific user messages. Option D is wrong because the model reads the system prompt directly.

2. A system prompt says "Never discuss competitor products." A user asks: "Compare your capabilities to GPT-4." What is the most likely behavior?

Claude receives a direct question that conflicts with system-level guidance.

B) Claude will attempt to honor the system prompt by avoiding direct competitor discussion while still being helpful. It typically redirects by discussing its own features. System prompts produce soft constraints that Claude navigates with nuance, not binary compliance.
