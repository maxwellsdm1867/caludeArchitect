Key Mechanisms

  • @import path/to/file.md — modular config, keeps CLAUDE.md DRY
  • .claude/rules/ — path-specific rules with glob patterns

Exam Rule User-level is NOT shared

"Teammate can't see my rules" → rules are in ~/.claude/CLAUDE.md. Move to project-level.

Merge Behavior

All levels merge (not replace). Avoid contradictions across levels.

4 — Plan vs Direct (TS 3.4)

Decision Tree

Multi-file + unclear approach → PLAN MODE
Investigation / unknown codebase → PLAN MODE
Architectural decisions → PLAN MODE
Single-file fix + clear trace → DIRECT
Implementing approved plan → DIRECT
Trivial change (typo, rename) → DIRECT

Plan-Then-Execute Pattern

  1. Plan mode: explore + propose approach
  2. Human review: approve/refine
  3. Direct execution: implement approved plan

Explore Subagent

Read-only exploration. Cannot modify files. Use for safe codebase understanding.

Trap Re-planning an approved plan

Once a plan is approved, execute directly. Re-entering plan mode risks second-guessing.

5 — Iterative Refinement (TS 3.5)

Technique Selection

ProblemTechnique
Wrong format/styleInput/output examples
Logic errorsTest-driven iteration
Unclear requirementsInterview pattern
Close but needs polishTargeted feedback

Test-Driven Iteration

  1. Write failing tests (= specification)
  2. Claude iterates until tests pass
  3. Add edge case tests, iterate again

Key Tests provide objective, automated success criteria. No manual review per iteration.

Interview Pattern

Instruct Claude to ask 3-5 clarifying questions before generating. Front-loads context gathering.

Trap Vague feedback

"Fix the email handling" is ambiguous. Provide specific error: "email field has 'JOHN@EXAMPLE.COM', expected 'john@example.com'."

6 — CI/CD Integration (TS 3.6)

Key Flags

FlagPurpose
-p "prompt"Headless mode (non-interactive)
--output-format jsonMachine-readable JSON output
--jsonShorthand for --output-format json
--output-format stream-jsonStreaming JSON
--continueResume most recent session

Session Isolation

-p mode starts fresh by default. Each CI run is isolated. Use --continue only when you need prior context.

CI Patterns

  • Per-file review — one claude -p per file, aggregate JSON findings
  • Multi-step — Step 1: analyze, Step 2: --continue to fix
  • Prior findings — capture JSON, inject into next prompt

Trap Inconsistent CI results

If results vary across runs, check for session state leakage. Ensure -p isolation and explicit prompt inputs.

8 — Distractor Detection Cheat Code

Common wrong answers on the exam. If you see these as options, eliminate them first:

Distractor PhrasingWhy It's WrongCorrect Direction
"Copy CLAUDE.md to each teammate's home directory" Doesn't scale, no sync, not version-controlled Project-level CLAUDE.md committed to git (3.1)
"Put a CLAUDE.md in every test directory" Duplication, maintenance nightmare, applies to non-test files too Single .claude/rules/ file with glob pattern (3.3)
